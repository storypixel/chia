Plain CSS Framework

Real CSS.
No build step.
No dependencies.

A component library that uses attribute selectors and CSS custom properties. One file. Readable styles. Works with any framework or none at all.

@import 'chia/chia.css'
Modules

Six files. Import what you need.

Each module is independent. Use them together or pull in only what you need.

tokens.css

Design tokens for typography, radius, color, spacing, and transitions. Uses Open Props for utility primitives.

reset.css

Modern CSS reset based on Andy Bell's approach. Sensible defaults without design opinions.

components.css

UI components styled via [data-slot] attribute selectors. Button, card, badge, input, avatar, and more.

utilities.css

Minimal layout utilities. Flex, grid, gap, padding, margin, sizing. Just the essentials.

grid.css

CSS Grid system with data-grid attributes. Auto-fill, fixed columns, named layouts, subgrid, container queries, and responsive breakpoints via data-sm / data-md / data-lg.

animations.css

Keyframes for fade, slide, scale, collapse, ping, shimmer, and spin. Framework-agnostic.

prose.css

Rich text typography via the .prose class. For editor output, markdown, and long-form content.

Usage

HTML with data attributes.

No class name soup. Components identify themselves with data-slot. Variants with data-variant.

<!-- Button -->
<button data-slot="button"
        data-variant="default"
        data-size="default">
  Click me
</button>

<!-- Card -->
<div data-slot="card">
  <div data-slot="card-header">Title</div>
  <div data-slot="card-content">Body</div>
</div>

<!-- Badge -->
<span data-slot="badge"
      data-variant="secondary">New</span>

<!-- Input -->
<input data-slot="input" placeholder="Search..." />

<!-- Grid -->
<div data-grid>              <!-- auto-fill responsive -->
  <div>Card 1</div>
  <div>Card 2</div>
  <div>Card 3</div>
</div>

<div data-grid="sidebar">     <!-- sidebar + content -->
  <nav>...</nav>
  <main>...</main>
</div>

<div data-grid="3" data-gap="lg">  <!-- fixed 3-col -->
  <div>...</div>
  <div data-grid-item="span-2">...</div>
</div>
Philosophy

Constraints that make CSS work.

One selector per rule

No compound selectors. No nesting. Every rule targets exactly one thing. Easy to find, easy to override.

No !important

If you need !important, your architecture is broken. Chia uses flat specificity so the cascade works for you.

Attributes over classes

data-slot identifies what something is. data-variant says how it looks. No BEM, no utility soup, no naming debates.

Custom properties everywhere

Every color, radius, size, and transition is a CSS variable. Theme by overriding tokens, not by forking components.

No build step

Import the CSS. That's it. No PostCSS, no Tailwind JIT, no config files. Works in a CDN link tag.

Framework agnostic

React, Vue, Svelte, plain HTML. Chia doesn't care. It styles elements, not components.

Standing on shoulders
